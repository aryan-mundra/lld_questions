📋 Logging System — Low Level Design

Learn by building it the way an interviewer expects: understand → gather requirements → design one requirement at a time → integrate → dry run.

1 · Understand the problem

Before any code, say the problem out loud in plain English. Build intuition first.

📝 Problem statement

Design a logging system. Code logs messages at different levels — DEBUG, INFO, WARN, ERROR — and each level should be handled appropriately (printed, formatted, maybe later written to a file or emailed). New levels or per-level behavior should be easy to add without rewriting one giant method.

ACTORS Who uses it?

  • Application code — calls logger.info(...) etc.
  • Logger — routes each message to the right handler.

ENTITIES What "things" exist?

  • Log level (DEBUG/INFO/WARN/ERROR)
  • Logger (the entry point)
  • A handler per level

BEHAVIOR What happens? (the flow)

logger.error("boom") → message travels a chain of handlers → the ERROR handler prints it

Each handler either handles the message (if it's their level) or passes it on. That "pass it down a chain" shape is the Chain of Responsibility.

🧩 Design pattern in focus

Chain of Responsibility — link handlers together; each decides to handle the message or forward it to the next. Adding a level = adding a handler; per-level behavior lives in its own class. Built in requirement NF1. (Beyond the core 8 patterns, but the classic one for logging.)

4 · Final integration

All pieces combined. Here's the class diagram, the chain, and the flow.

📦 Class diagram

«enum» LogLevel
DEBUG, INFO,WARN, ERROR
LogProcessor «abstract»
- next : LogProcessor
+ log(level, msg)↑ Debug/Info/Warn/Error
Logger «facade»
- chain : LogProcessor
+ debug/info/warn/error(msg)

🔗 The chain

Logger → [DebugProcessor] → [InfoProcessor] → [WarnProcessor] → [ErrorProcessor] → null
// each: "is this MY level? handle it. else → pass to next"

🔗 Relationships (in plain words)

──▷
IS-A: Debug/Info/Warn/ErrorProcessor is a LogProcessor.
◇──
NEXT: each LogProcessor holds the next one — the links that form the chain.
- -▷
USES: Logger builds the chain once and feeds every message into its head.

🌊 End-to-end flow

logger.error("boom")
  └─ Debug: not mine → pass
  └─ Info:  not mine → pass
  └─ Warn:  not mine → pass
  └─ Error: MINE → print "[ERROR] boom" ✅
